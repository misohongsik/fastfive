Fastfive 웹사이트 프론트엔드 디버깅 및 점검 리스트입니다.

현재 프로젝트는 Next.js 16, React 19, Tailwind CSS v4 등 최신 (또는 실험적) 스택을 사용하고 있어 호환성 및 설정 점검이 최우선입니다.

🔍 1. CSS 및 스타일링 (Tailwind v4)
Unknown at rule @theme 경고: 기능상 문제는 없으나, VS Code가 Tailwind v4의 새로운 문법을 인식하지 못해 발생합니다.
점검: .vscode/settings.json에 css.customData 설정이 있는지, 또는 files.associations에 *.css: tailwindcss 설정이 되어 있는지 확인하세요.
PostCSS 플러그인: postcss.config.mjs에 @tailwindcss/postcss가 올바르게 설정되어 있습니다. (OK)
📦 2. 의존성 및 버전 호환성 (Critical)
Next.js 16 & React 19: 현재 package.json에 명시된 버전이 매우 높습니다(Next 16은 아직 정식 출시 전일 가능성이 큼).
점검: npm list next react로 실제 설치된 버전을 확인하고, 실행 시 에러가 발생한다면 Stable 버전(Next 15, React 19)으로 조정을 고려하세요.
라이브러리 충돌: gsap, lenis, react-hook-form 등이 React 19와 호환되는지 확인이 필요합니다.
⚡ 3. 렌더링 및 성능 (Hydration)
HTML 구조 위반: React 19는 Hydration 에러에 더 엄격할 수 있습니다. <div> 안에 <p>를 넣거나 하는 시멘틱 오류가 없는지 확인하세요.
클라이언트 컴포넌트: SmoothScroller.tsx 등 브라우저 API(window, document)를 쓰는 컴포넌트 최상단에 "use client"가 명시되어 있는지 확인하세요.
🖼️ 4. 이미지 및 폰트 최적화
폰트 로딩: globals.css에서 @font-face를 직접 사용 중입니다.
권장: Next.js의 next/font/local 기능을 사용하여 Layout Shift(CLS)를 방지하는 것이 좋습니다.
이미지: <img> 태그 대신 next/image를 사용하여 자동 포맷 변환(WebP) 및 리사이징 혜택을 받고 있는지 점검하세요.
